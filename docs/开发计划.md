## 二、关卡设计

| 关卡 | 网格 | 数字数 | 挖空比例 | 备注 |
|------|------|--------|---------|------|
| Level 1 | 4×4 | 16 | 0% | 教学关，不消耗体力 |
| Level 2 | 6×6 | 36 | 10% | 过渡关 |
| Level 3 | 8×8 | 64 | 20% | 正式关开始 |
| Level 4–19 | 8×8 | 64 | 线性递增 | 每关约 +2.4% |
| Level 20 | 8×8 | 64 | 60% | 最终关 |

**挖空比例公式：** `blankRatio = 0.20 + (level - 3) × (0.40 / 17)`

**关卡生成算法（Warnsdorff 启发）：**
1. 在 N×N 网格上生成哈密顿路径（所有格子都在路径上，无空格）
2. 路径上第 i 个格子赋值 = i
3. 随机选取 `floor(total × blankRatio)` 个非首尾节点设为挖空（显示为灰色虚线圆）
4. 输出 `LevelData` 供渲染使用

## 三、核心规则确认

### 操作方式
- **拖拽**：按住节点 k → 拖动手指到 k+1 → 松手触发校验
- 拖拽过程中实时绘制半透明预览线
- 松手校验通过 → 绿色实线锁定路径
- 松手校验失败 → 红色闪烁 + 屏幕震动 + 扣1生命

### 星级标准
| 通关时剩余生命 | 星级 |
|-------------|------|
| 3颗心 | ⭐⭐⭐ |
| 2颗心 | ⭐⭐ |
| 1颗心 | ⭐ |

### 体力与生命
- **体力**：初始5（满），上限5，每20分钟+1；教学关不消耗，Level 2起每次进关-1
- **生命**：初始3条，错误连接-1，归零触发失败弹窗，每日重置
- **0生命入关**：弹提示「今日生命已耗尽」，阻止进入，不扣体力

### 道具逻辑
- **提示（💡）**：高亮 k+1 节点 + 画出 k→k+1 一段线（仅下一步）
- **范围提示（💧）**：高亮当前节点周围8格，半透明绿色区域
- **放大镜（🔍）**：局部放大当前区域（Camera zoom 实现）
- 库存 > 0 → 按钮显示「使用」直接扣库存；库存 = 0 → 显示「▶ 观看广告」

### 签到规则
- 7天循环，严格连续：断签一天，当周进度从第1天重置
- 奖励：Day1=2体力，Day2=提示x1，Day3=范围提示x1，Day4=2体力，Day5=提示x1，Day6=范围提示x1，Day7=3体力+提示x2

### 20关全通后
- 弹出全关卡通关成就界面
- 之后所有关卡可重玩刷星（正常消耗体力）

## 五、数据模型

```typescript
interface PlayerData {
  // 首次启动
  privacyAccepted: boolean;
  language: 'en' | 'ja';

  // 关卡进度
  unlockedLevel: number;                      // 已解锁最大关卡（初始=1）
  levelStars: Record<number, number>;         // { 3: 2 } = 第3关2星

  // 体力（上限5，每20分钟+1）
  stamina: number;
  staminaLastRecoverTime: number;             // 上次恢复时间戳 ms

  // 生命（初始3，每日重置）
  lives: number;
  livesResetDate: string;                     // 'YYYY-MM-DD'

  // 道具库存
  items: {
    hint: number;
    rangeHint: number;
  };

  // 七日签到
  signIn: {
    weekStartDate: string;                    // 本周期开始日期 'YYYY-MM-DD'
    claimedDays: number[];                    // 已领取天数 [1, 2, 3]
    lastSignInDate: string;                   // 上次签到日期
  };

  // 设置
  settings: {
    bgmOn: boolean;
    sfxOn: boolean;
  };
}
```

#### InputHandler
- [ ] 触摸按下：识别命中的节点（按格子坐标映射）

#### GameController
- [ ] 校验逻辑：目标是否 = current+1 且在8方向相邻

- [ ] 生命归零：触发 FailDialog
- [ ] 全部连完：计算星级，触发 WinDialog
- [ ] 道具逻辑：
  - 提示：高亮 k+1 节点 + 画 k→k+1 辅助线（3秒后消失）
